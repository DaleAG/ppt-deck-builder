# PPT Workflow

## Goal
Turn a topic into a clear, persuasive, editable presentation deck.

## Step 1: Align the brief
Confirm these before writing slides:
- audience
- goal
- style
- page-count range
- whether to discuss first or execute directly

Default to discuss-first when the user wants control over direction.

## Step 2: Build the narrative spine
Do not start with page text.
Compress the whole deck into 3-5 core questions or statements.
Examples:
- What is it?
- Why does it matter now?
- What value does it create?
- What should we do next?

## Step 3: Build the evidence pool
Before page writing, collect:
- real cases
- proof points
- source material
- comparisons
- boundaries and objections

Organize evidence by value, not just by source.

## Step 4: Design the middle of the deck around proof
The middle section should usually follow:
- real case / concrete example
- what it proves
- what principle it suggests

Avoid leading with abstract principles and only later attaching examples.

## Step 5: Draft page structure before polishing
Recommended order:
- cover
- positioning / what it is
- why now / why hot
- differentiation
- proof / core cases
- value
- boundaries
- recommendation / next step
- closing page if needed

## Step 6: Use versioned rebuilds
If the draft is structurally off:
- do not over-patch blindly
- open v2 / v3 and rebuild cleanly

## Step 7: Refine visuals after structure is stable
Adjust only after the story is correct:
- theme color
- logo
- emphasis cards
- thank-you page
- brand assets
