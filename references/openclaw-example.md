# Worked Example: OpenClaw Deck

## Project type
Boss-facing and internal-team presentation deck.

## Goal
Help people who have not used OpenClaw quickly understand:
- what it is
- why it is hot
- what value it creates
- how to embrace it

## What worked
- Reframed the whole deck around a four-question spine
- Pulled the deck back from generic Agent preaching to OpenClaw-specific positioning
- Used real cases as the backbone
- Treated the PPT itself as a proof case
- Switched from patching v1 to rebuilding v2/v3
- Used HTML/CSS as the main production format

## Key lessons
- If the subject is a specific product, make that product the subject again
- Missing cases should be merged back into the narrative, not pasted as classification pages
- Boss-facing decks usually benefit from statement titles
- Visual upgrades should come after narrative stability
- PDF export needs dedicated print rules
