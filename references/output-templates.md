## 2. Case page template
### Title
Use a clear statement title.

### One-line conclusion
State what the case proves.

### Recommended layout
Left side:
- the case itself
- who did what
- what happened

Right side:
- what this case proves
- why it matters
- what boundary or caution exists

### Footer sentence
Summarize the takeaway in one sentence that can be spoken aloud.

## 3. Value summary page template
### Title
What real value does X create?

### Structure
- value 1
- value 2
- value 3
- value 4

### Rule
Each value must be expressed as a real organizational or workflow effect, not abstract praise.

## 4. Boundary page template
### Title
X is worth trying, but its boundaries must be understood.

### Recommended sections
- threshold / setup cost
- stability limitations
- real cost
- risk and permissions

### Rule
Boundary pages should reduce hype without killing momentum.

## 5. Recommendation page template
### Title
How should we embrace X?

### Structure
- first understand it
- start with small, high-value pilots
- form team methods and rules while using it

### Rule
Recommendation pages should feel actionable, not preachy.

## 6. Closing page template
Options:
- Thank you page
- one-line final recommendation
- Q&A page

If a human presenter identity matters, include author/signature and optional avatar.
