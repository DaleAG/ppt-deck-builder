# PPT Deck Builder

Use this skill to turn a vague presentation goal into a structured, editable, boss-facing deck.

## Quick Start

1. Read `references/ppt-workflow.md` for the end-to-end working sequence.
2. Read `references/story-architecture.md` when the deck needs a strong storyline instead of a pile of pages.
3. Read `references/cases-and-evidence.md` when the topic needs examples, proof points, or case layering.
4. Read `references/html-prototyping-and-export.md` before building HTML/CSS slides or exporting PDF.
5. Read `references/input-brief-template.md` when you need to structure the user brief or intake.
6. Read `references/output-templates.md` when you need reusable page patterns or deck skeletons.
7. Read `references/openclaw-example.md` only when you want a concrete worked example of how this playbook was used on a real AI product deck.

## Core Rules

- Align audience, goal, style, page count, and collaboration mode before writing pages.
- Build the narrative spine before filling slide copy.
- Use evidence and cases before abstract principles.
- When a draft drifts, fix the structure before polishing the visuals.
- If the deck is for bosses or internal teams, prefer statement-style titles over too many question titles.
- Do not let chat phrasing leak into slide copy.
- When the user says some important examples were lost, merge them back into the narrative instead of pasting a taxonomy page.
- Prefer editable delivery formats when iteration speed matters.

## Recommended Delivery Order

1. Goal alignment
2. Narrative spine
3. Case / evidence pool
4. Page-by-page outline
5. Draft copy
6. Editable prototype
7. Visual refinement
8. Export / packaging
9. Method capture for reuse
